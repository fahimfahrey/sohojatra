# PWA Features Documentation

## Overview

Sohojatra now includes full Progressive Web App (PWA) capabilities, allowing users to install the app on their devices and use it like a native application.

## Features Implemented

### 1. **App Installation**

- Users can install the app on their devices (mobile and desktop)
- Install prompt appears automatically after 3 seconds (can be dismissed)
- Install button available in browser menu
- Works on Chrome, Edge, Safari (iOS 16.4+), and other modern browsers

### 2. **Offline Support**

- App shell cached for offline access
- Static assets (JS, CSS, images) cached automatically
- Map tiles cached for offline viewing (OpenStreetMap & OpenTopoMap)
- Google Fonts cached for consistent typography

### 3. **App Manifest**

- **Name**: Sohojatra - Ride Sharing
- **Short Name**: Sohojatra
- **Theme Color**: #3b82f6 (Blue)
- **Display Mode**: Standalone (full-screen app experience)
- **Icons**: 192x192 and 512x512 PNG icons

### 4. **Service Worker**

- Automatic updates when new versions are deployed
- Background sync for notifications
- Push notification support (already implemented)
- Efficient caching strategies:
  - **CacheFirst** for map tiles (30 days)
  - **CacheFirst** for fonts (1 year)
  - **NetworkFirst** for API calls (default)

### 5. **Install Prompt Component**

- Custom UI for installation
- Dismissible with "Not now" option
- Remembers user preference (won't show again if dismissed)
- Responsive design for mobile and desktop

## User Experience

### Installation Process

#### On Mobile (Android/iOS):

1. Visit the app in a browser
2. After 3 seconds, an install prompt appears at the bottom
3. Tap "Install" to add to home screen
4. Or use browser menu → "Add to Home Screen"

#### On Desktop (Chrome/Edge):

1. Visit the app in a browser
2. Look for the install icon in the address bar
3. Click to install
4. App opens in its own window

### Benefits for Users

- **Faster Load Times**: Cached assets load instantly
- **Offline Access**: View cached content without internet
- **Native Feel**: Full-screen experience without browser UI
- **Quick Access**: Launch from home screen/app drawer
- **Smaller Size**: No app store download required
- **Auto Updates**: Always get the latest version

### Files Added/Modified

- `PWA_FEATURES.md` - This documentation

#### Modified Files:

- `vite.config.ts` - Added VitePWA plugin configuration
- `index.html` - Added PWA meta tags and manifest link
- `src/App.tsx` - Added PWAInstallPrompt component
- `src/main.tsx` - Added service worker registration
- `package.json` - Added vite-plugin-pwa dependency

### Configuration

#### Vite PWA Plugin (`vite.config.ts`):

```typescript
VitePWA({
  registerType: "autoUpdate",
  includeAssets: ["sohojatra_ico.png", "banner_image.png", "sohojatra.png"],
  manifest: {
    /* ... */
  },
  workbox: {
    runtimeCaching: [
      /* ... */
    ],
  },
});
```

#### Caching Strategy:

- **Map Tiles**: CacheFirst, 500 entries, 30 days
- **Fonts**: CacheFirst, 10 entries, 1 year
- **Static Assets**: Precached during installation
- **API Calls**: NetworkFirst (default)

### Service Workers

The app now uses TWO service workers:

1. **Workbox SW** (auto-generated by vite-plugin-pwa): Handles caching and offline support
2. **Custom SW** (`public/sw.js`): Handles push notifications

Both work together seamlessly without conflicts.

## Customization

### Change Theme Color:

Edit `vite.config.ts`:

```typescript
manifest: {
  theme_color: "#your-color",
  background_color: "#your-color"
}
```

### Change App Name:

Edit `vite.config.ts`:

```typescript
manifest: {
  name: "Your App Name",
  short_name: "Short Name"
}
```

### Adjust Cache Duration:

Edit `vite.config.ts` → `workbox.runtimeCaching`:

```typescript
expiration: {
  maxAgeSeconds: 60 * 60 * 24 * 30; // 30 days
}
```

### Disable Install Prompt:

Remove `<PWAInstallPrompt />` from `src/App.tsx`

## Troubleshooting

### Install Prompt Not Showing:

- Check if app is already installed
- Clear browser cache and reload
- Ensure HTTPS is enabled (required for PWA)
- Check browser console for errors

### Offline Mode Not Working:

- Build and preview the app (dev mode has limited SW support)
- Check Network tab in DevTools → Offline checkbox
- Verify service worker is registered (Application tab → Service Workers)

### Updates Not Applying:

- Service worker updates automatically
- Force update: Unregister SW in DevTools → Reload
- Or clear site data and reload
